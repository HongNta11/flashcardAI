# Flashcard App Design

**Date:** 2026-04-19  
**Status:** Approved

## Overview

A personal flashcard study app with three components:
1. A Claude Code skill that generates flashcards from Markdown books
2. A FastAPI backend that serves cards and stores progress
3. A Preact PWA for reviewing flashcards on iOS

---

## Architecture

```
books/
  clean-code.md
  clean-code.json        ← generated by Claude skill
  pragmatic-programmer.md
  pragmatic-programmer.json

Claude Code skill (/generate-flashcards)
  → reads .md file
  → calls Claude API to produce Q + 4 options + explanation
  → writes .json alongside the .md

FastAPI backend
  → GET /books           (list books with cards)
  → GET /books/:id/cards (fetch cards for a book)
  → POST /progress       (save user's card result)
  → GET /progress/:book  (fetch progress for a book)
  → token auth header on all routes

PWA (Preact + HTM, no bundler)
  → Book list screen → Book quiz screen
  → Multiple choice card → reveal explanation on select
  → IndexedDB for offline card caching
  → Syncs progress to server when online
```

---

## Claude Code Skill

**Command:** `/generate-flashcards <book.md>`  
Running without args lists available `.md` files in the books folder.

**Behavior:**
- Reads the target `.md` file and chunks it into sections
- Calls Claude API to generate 3–5 cards per section, each with:
  - A question
  - 4 options (1 correct + 3 Claude-generated distractors)
  - An explanation shown after the user answers
- Writes output to `<book-name>.json` in the same folder
- Re-running overwrites the existing JSON (full regeneration)

## Flashcard JSON Format

```json
{
  "book": "clean-code",
  "generated_at": "2026-04-19T10:00:00Z",
  "cards": [
    {
      "id": "cc-001",
      "question": "What is the main purpose of a function?",
      "options": [
        "To do one thing and do it well",
        "To handle all related tasks together",
        "To minimize the number of files",
        "To maximize code reuse"
      ],
      "correct_answer": "To do one thing and do it well",
      "explanation": "A function should do one thing only — this makes it easier to name, test, and reason about."
    }
  ]
}
```

- `correct_answer` stores the text of the correct option (robust to frontend shuffling)
- Frontend shuffles the `options` array on display, then compares selected text to `correct_answer`

## Backend (FastAPI + SQLite)

**Endpoints:**

| Method | Path | Description |
|--------|------|-------------|
| GET | `/books` | List books that have a `.json` file, include card count and user progress % |
| GET | `/books/:id/cards` | Return all cards for a book |
| POST | `/progress` | Save a single card result `{book_id, card_id, correct}` |
| GET | `/progress/:book` | Return per-card progress for a book |

**Auth:** Bearer token in `Authorization` header. Single static token for personal use, configured via environment variable.

**Progress table (SQLite):**
```sql
CREATE TABLE progress (
  user_token  TEXT,
  book_id     TEXT,
  card_id     TEXT,
  correct     BOOLEAN,
  reviewed_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

Progress % per book = distinct card_ids reviewed / total cards in JSON.

## PWA (Preact + HTM)

### Book List Screen
- Lists all books with a generated `.json` file
- Each entry shows: book title, total card count, progress % reviewed
- Tap to enter quiz mode for that book

### Book Quiz Screen
- One card at a time, shuffled order each session
- Progress bar: "12 / 48 cards"
- Question text at top
- 4 option buttons below
- On selection:
  - Correct: button highlights green, explanation appears below
  - Wrong: selected button highlights red, correct button highlights green, explanation appears
- "Next Card" button to advance

### End of Deck Screen
- Score: "X correct out of Y"
- Buttons: "Review Again" or "Back to Books"

### Offline Support
- Cards cached in IndexedDB on first load
- Progress synced to server when online; queued locally when offline

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Card generation | Claude Code skill + Claude API |
| Backend | Python 3.12, FastAPI, SQLite, Uvicorn |
| Frontend | Preact 10 + HTM, no bundler, vanilla JS |
| Storage | JSON files (cards), SQLite (progress), IndexedDB (offline cache) |
| Auth | Static bearer token |

---

## Out of Scope

- Multiple user accounts (single token only)
- Editing cards via UI (edit the JSON directly)
- Support for non-Markdown book formats (MD first)
- Spaced repetition algorithm (simple shuffle for now)
